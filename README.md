# SQL_repl_scaling
## 1. Task
## Характеристики:        Master-Slave                              Master-Master
## Операции записи:       Только на Master;                         На любом узле Master;
## Операции чтения:       На всех узлах;                            На всех узлах; 
## Топология:             Один лидер, однонаправленная репликация;  Несколько лидеров, двунаправленная репликация;
## Обработка конфликтов:  Не требуется,т.к один источник записи;    Критически важна и сложна;
## Согласованность:       Сильная согласованность;                  Сложно поддерживать;
## Отказоустойчивость:    Требует повышения Slave до роли  Master;  Часто происходит незаметно, другие Master обрабатывают трафик;
## Сложность:             Проще в настройке и управлении;           Значительно сложнее.
